Proposed AWS Services

  • Amazon Elastic Compute Cloud (EC2)
    • Host the Flask application serving the REST API.
    • Choose an appropriate instance type based on your expected traffic and resource requirements.
  • Amazon Relational Database Service (RDS)
    • Manage the PostgreSQL database for storing weather data and statistics.
    • Select a suitable instance type and storage size.
    • Configure database settings and security.

    OR

  • Amazon NoSQL DB like Dynamodb / DocumentDB
    • Manage the Schema with each station or year as Nodes & values related as edges
      • This is depended also on queries performed on the DB
    • Configure database settings and security.

    OR

  • Create External Tables on top of S3 files
    • If we have so much data intake , we can perform ETLs & write them as delta format this will help in faster queries
  • AWS Lambda
    • Create a Lambda function to handle the data ingestion task.
    • Trigger the Lambda function using AWS CloudWatch Events on a schedule (e.g., daily, hourly).
    • The Lambda function can use the Boto3 library to interact with S3 (where the weather data files are stored) and the RDS database.
  • Amazon Simple Storage Service (S3)
    • Store the raw weather data files.
    • Configure S3 bucket permissions to allow the Lambda function to access the files.
